# WildGuard AI Oracle DB 설정 문서

## 1. 목적

Oracle DB는 WildGuard AI에서 로그인 사용자 정보와 위험도 예측 기록을 저장하기 위해 사용한다.

현재 프로젝트에서 Oracle DB가 필요한 기능은 다음과 같다.

| 기능 | DB 필요 여부 | 설명 |
| --- | --- | --- |
| 회원가입 | 필요 | 사용자 계정 저장 |
| 로그인 | 필요 | 사용자 계정 조회 |
| 위험도 예측 | 선택 | 예측 자체는 가능하지만, 로그인 사용자의 기록 저장에는 DB 필요 |
| 예측 기록 조회 | 필요 | 저장된 위험도 예측 기록 조회 |
| 이미지/영상 탐지 | 불필요 | 현재는 파일 업로드/결과 이미지 저장 중심 |
| 대응 상담 챗봇 | 불필요 | 현재는 위험도 예측 결과 기반 응답 생성 |

---

## 2. PC별 Oracle 실행 방식

집 PC와 학원 PC는 Oracle 접속 환경이 다를 수 있다.

따라서 `config/db_config.py`에 특정 IP나 Oracle Client 경로를 직접 고정하지 않고, 실행 전에 CMD 환경변수로 DB 접속 정보를 설정한다.

---

## 3. 집 PC 실행 예시

집 PC에서 Oracle 리스너가 `192.168.219.103:1521`처럼 로컬 IP로 열려 있는 경우 아래처럼 실행한다.

```cmd
cd /d E:\Projects\wildguard-ai
.venv\Scripts\activate

set DB_HOST=192.168.219.103
set DB_PORT=1521
set DB_SERVICE_NAME=XEPDB1
set USE_ORACLE_THICK_MODE=false

python app.py
```

만약 `XEPDB1`로 접속이 실패하면 `XE`로 한 번 더 테스트한다.

```cmd
set DB_SERVICE_NAME=XE
python app.py
```

---

## 4. 학원 PC 실행 예시

학원 PC에서 Oracle이 `localhost:1521`로 열려 있고 Thick Mode가 필요한 경우 아래처럼 실행한다.

```cmd
cd /d E:\Projects\wildguard-ai
.venv\Scripts\activate

set DB_HOST=localhost
set DB_PORT=1521
set DB_SERVICE_NAME=XE
set USE_ORACLE_THICK_MODE=true
set ORACLE_CLIENT_DIR=C:\oraclexe\instantclient_19_25

python app.py
```

학원 PC의 Oracle Client 경로가 다르면 `ORACLE_CLIENT_DIR` 값은 실제 설치 경로에 맞게 수정한다.

## 5. Oracle 서비스 상태 확인

DB 접속 오류가 발생하면 먼저 Oracle 서비스가 설치되어 있는지 확인한다.

```cmd
sc query state= all | findstr /I "Oracle"
```

일반적으로 아래 서비스가 보여야 한다.

```text
OracleServiceXE
OracleOraDB21Home1TNSListener
```

서비스가 꺼져 있으면 관리자 권한 CMD에서 실행한다.

```cmd
net start OracleServiceXE
net start OracleOraDB21Home1TNSListener
```

이미 켜져 있으면 아래 메시지가 나와도 정상이다.

```text
요청한 서비스가 이미 시작되었습니다.
```

---

## 6. Oracle 리스너 포트 확인

Oracle 서비스가 켜져 있어도 리스너가 어떤 주소로 열렸는지 확인해야 한다.

```cmd
netstat -ano | findstr 1521
```

예시 1: `localhost` 계열로 열리는 경우

```text
TCP    127.0.0.1:1521    0.0.0.0:0    LISTENING
```

이 경우 `DB_HOST=localhost` 또는 `DB_HOST=127.0.0.1`을 사용할 수 있다.

예시 2: 로컬 네트워크 IP로 열리는 경우

```text
TCP    192.168.219.103:1521    0.0.0.0:0    LISTENING
```

이 경우 `DB_HOST=192.168.219.103`처럼 실제 표시된 IP를 사용한다.

## 7. 자주 발생하는 오류

### 7.1 DPY-6005 / WinError 10061

오류 예시:

```text
DPY-6005: cannot connect to database
[WinError 10061] 대상 컴퓨터에서 연결을 거부했으므로 연결하지 못했습니다
```

의미:

```text
Flask 서버가 Oracle DB에 접속하려 했지만,
Oracle 서비스 또는 리스너에 연결하지 못한 상태
```

확인 순서:

```cmd
sc query state= all | findstr /I "Oracle"
netstat -ano | findstr 1521
```

그다음 `netstat`에 표시된 주소를 기준으로 `DB_HOST`를 설정한다.

---

### 7.2 localhost로 접속 실패

`netstat` 결과가 아래처럼 나오는 경우:

```text
TCP    192.168.219.103:1521    0.0.0.0:0    LISTENING
```

그런데 코드 또는 환경변수에서 `DB_HOST=localhost`를 사용하면 접속 실패가 날 수 있다.

해결:

```cmd
set DB_HOST=192.168.219.103
python app.py
```

---

### 7.3 서비스명 XE / XEPDB1 혼동

Oracle XE 환경에 따라 서비스명이 다를 수 있다.

우선순위:

```text
집 PC: XEPDB1 우선 테스트
학원 PC: XE 우선 테스트
```

접속 실패 시 둘 중 다른 값으로 바꿔 테스트한다.

```cmd
set DB_SERVICE_NAME=XEPDB1
python app.py
```

```cmd
set DB_SERVICE_NAME=XE
python app.py
```

## 8. 주요 테이블

### 8.1 USERS

사용자 계정 정보를 저장한다.

```sql
CREATE TABLE users (
    user_id NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    username VARCHAR2(100) NOT NULL UNIQUE,
    password_hash VARCHAR2(255) NOT NULL,
    name VARCHAR2(100),
    organization VARCHAR2(100),
    user_type VARCHAR2(50),
    created_at DATE DEFAULT SYSDATE
);
```

프로젝트 코드의 실제 컬럼명이 다르면 현재 DB 스키마에 맞게 조정한다.

---

### 8.2 RISK_PREDICTION_LOGS

로그인 사용자의 위험도 예측 기록을 저장한다.

```sql
CREATE TABLE risk_prediction_logs (
    log_id NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    user_id NUMBER,
    region_name VARCHAR2(100),
    location VARCHAR2(100),
    weather VARCHAR2(50),
    time_zone VARCHAR2(50),
    season VARCHAR2(50),
    predicted_score NUMBER,
    risk_grade VARCHAR2(20),
    main_risk_species VARCHAR2(100),
    message CLOB,
    created_at DATE DEFAULT SYSDATE,
    CONSTRAINT fk_risk_user FOREIGN KEY (user_id) REFERENCES users(user_id)
);
```

## 9. 운영 원칙

- DB 접속 정보는 GitHub에 업로드하지 않는다.
- `.env` 파일은 `.gitignore`에 포함하고 커밋하지 않는다.
- PC별 Oracle 접속 정보는 CMD 환경변수로 설정한다.
- 집 PC IP를 `config/db_config.py`에 직접 박아서 커밋하지 않는다.
- 학원 PC와 집 PC를 오갈 때는 실행 전 `DB_HOST`, `DB_SERVICE_NAME`, `USE_ORACLE_THICK_MODE`를 다시 확인한다.
- 예측 기록 저장 실패가 위험도 예측 API 전체 실패로 이어지지 않도록 처리한다.
