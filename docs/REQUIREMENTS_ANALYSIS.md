# WildGuard AI 요구사항 분석서

## 1. 프로젝트 개요

### 프로젝트명

WildGuard AI

### 주제

서울시 멧돼지 민원신고 현황 기반 야생동물 출현 위험도 예측 및 대응 지원 AI 서비스

### 목적

WildGuard AI는 야생동물 출현 위험을 데이터 기반으로 분석하고, 사용자에게 위험도 예측·이미지 탐지·대응 상담을 제공하는 AI 서비스이다.

| 차수 | 프로젝트 유형 | 핵심 내용 |
|---|---|---|
| 1차 | 정형 데이터 ML | 서울시 멧돼지 민원신고 데이터를 활용한 자치구·월별 위험도 예측 |
| 2차 | Vision / YOLO | AI Hub 원천 이미지와 bbox 라벨을 활용한 야생동물 탐지 |
| 3차 | LLM / RAG | 위험도 예측 결과와 탐지 결과 기반 야생동물 대응 상담 |
| 4차 | SLM | 현장 환경에서 사용할 수 있는 경량 대응 상담 모델 |

### 구성 이유

1차 ML은 실제 신고 현황 데이터를 활용하여 지역과 월에 따른 출현 위험도를 예측한다. 2차 Vision은 AI Hub 이미지와 bbox 라벨을 활용하여 실제 사진 속 야생동물을 탐지한다. 이렇게 역할을 분리하면 1차는 지역·시기 기반 예측, 2차는 이미지 기반 탐지로 자연스럽게 연결된다.

## 2. 문제 정의

야생동물 출현은 농장, 산림, 도로, 마을 주변에서 안전 문제와 재산 피해를 유발할 수 있다. 특히 멧돼지는 도심 인근 산림과 주거지 경계에서 출현할 수 있으며, 사용자가 직접 위험도를 판단하기 어렵다.

| 문제 | 설명 |
|---|---|
| 위험 판단 어려움 | 지역과 시기에 따른 출현 위험도를 직접 판단하기 어렵다. |
| 이미지 확인 한계 | 현장 사진만으로 야생동물 객체 여부를 빠르게 판단하기 어렵다. |
| 대응 정보 부족 | 야생동물 발견 시 행동 요령을 알기 어렵다. |
| 기록 관리 부족 | 출현 예측과 대응 기록을 체계적으로 관리하기 어렵다. |

## 3. 사용자 정의

| 사용자 | 설명 | 주요 니즈 |
|---|---|---|
| 농장주 | 농작물 피해 예방이 필요한 사용자 | 야간 출현 위험 확인, 대응 방법 안내 |
| 산림관리원 | 산림·등산로 안전을 관리하는 사용자 | 야생동물 탐지 및 위험 상황 판단 |
| 지자체 담당자 | 주민 신고와 지역 안전을 관리하는 사용자 | 위험도 기록, 대응 안내, 관리 자료 확보 |
| 일반 사용자 | 야생동물을 발견한 사용자 | 안전한 행동 요령 확인 |

## 4. 데이터 요구사항

### 4.1 1차 ML 데이터

| 항목 | 내용 |
|---|---|
| 원본 데이터 | 서울시 멧돼지 민원신고 현황 CSV |
| 대상 경로 | `data/seoul_wildlife/seoul_boar_reports.csv` |
| 행 수 | 360행 |
| 기간 | 2022년 ~ 2024년 |
| 자치구 수 | 11개 |
| 주요 컬럼 | 연도, 자치구, 월, 출현개체수, 포획개체수 |

### 4.2 1차 ML Feature

| 구분 | Feature |
|---|---|
| 기본 feature | 연도, 자치구, 월, 계절 |
| 파생 feature | district_total_count |
| 파생 feature | district_avg_count |
| 파생 feature | month_avg_count |
| 파생 feature | district_month_avg_count |
| 파생 feature | district_total_capture |

### 4.3 위험도 라벨 생성 기준

| 조건 | risk_level |
|---|---|
| 출현개체수 0건 | low |
| 출현개체수 1건 | medium |
| 출현개체수 2건 이상 | high |

### 4.4 데이터 사용 이유

AI Hub 이미지는 야생동물 객체 탐지에는 적합하지만, 출현하지 않은 경우와 비교하는 지역별 출현 위험도 예측에는 한계가 있다. 따라서 1차 ML은 실제 민원신고 현황 기반 데이터로 구성한다.

## 5. 기능 요구사항

| ID | 요구사항 | 설명 |
|---|---|---|
| RQ-001 | 서울시 멧돼지 신고 데이터 구성 | CSV를 프로젝트 데이터 폴더에 배치한다. |
| RQ-002 | 위험도 라벨 생성 | 출현개체수 기준 low / medium / high를 생성한다. |
| RQ-003 | 파생 feature 생성 | 자치구·월별 과거 출현 통계를 생성한다. |
| RQ-004 | 위험도 예측 모델 학습 | RandomForest 기반 위험도 분류 모델을 학습한다. |
| RQ-005 | 위험도 예측 화면 제공 | 사용자가 자치구와 월을 입력해 위험도를 확인한다. |
| RQ-006 | 위험도 예측 API 제공 | `/api/risk/predict`에서 예측 결과를 반환한다. |
| RQ-007 | YOLO 라벨 변환 | AI Hub JSON bbox를 YOLO txt 형식으로 변환한다. |
| RQ-008 | 이미지 탐지 API | 이미지 업로드 시 야생동물을 탐지한다. |
| RQ-009 | 상담 챗봇 | 위험도와 탐지 결과를 바탕으로 대응 방법을 안내한다. |
| RQ-010 | 결과 저장 | 로그인 사용자 예측 결과를 Oracle DB에 저장한다. |
