# AI기반 야생동물 위험도 예측 및 대응 서비스

## 1. 프로젝트 개요

### 주제
AI Hub 야생동물 라벨링 JSON 메타데이터 기반 야생동물 출현 위험도 예측 및 대응 지원 AI 서비스

### 목적
WildGuard AI는 야생동물 출현 상황을 데이터 기반으로 분석하고, 사용자에게 위험도 예측·이미지 탐지·대응 상담을 제공하는 AI 서비스이다.

| 차수 | 프로젝트 유형 | 핵심 내용 |
|---|---|---|
| 1차 | 정형 데이터 ML | AI Hub 라벨링 JSON 메타데이터를 CSV로 변환하여 위험도 예측 |
| 2차 | Vision / YOLO | 원천 이미지와 bbox 라벨을 활용한 사족보행 야생동물 탐지 |
| 3차 | LLM / RAG | 위험도 예측 결과와 탐지 결과 기반 야생동물 대응 상담 |
| 4차 | SLM | 현장 환경에서 사용할 수 있는 경량 대응 상담 모델 |

### 구성 이유
같은 AI Hub 데이터셋을 사용하지만 1차와 2차의 역할을 분리한다. 1차는 이미지 자체가 아니라 JSON의 `day`, `weather`, `location`, `GPS`, `object_count`, `bbox_area_ratio` 같은 메타데이터를 표 데이터로 변환하여 머신러닝 위험도 예측에 사용한다. 2차는 원천 이미지와 bbox를 사용하여 YOLO 객체 탐지 모델을 학습한다.

## 2. 문제 정의

야생동물 출현은 농장, 산림, 도로, 마을 주변에서 안전 문제와 재산 피해를 유발할 수 있다. 특히 멧돼지, 고라니, 너구리 등 사족보행 야생동물은 야간에 출현하는 경우가 많고, 사용자가 직접 위험도를 판단하기 어렵다.

| 문제 | 설명 |
|---|---|
| 위험 판단 어려움 | 사용자가 출현 상황의 위험도를 직접 판단하기 어렵다. |
| 이미지 확인 한계 | 현장 사진만으로 야생동물 객체 여부를 빠르게 판단하기 어렵다. |
| 대응 정보 부족 | 동물 종류, 시간대, 상황에 따른 대응 방법을 알기 어렵다. |
| 데이터 활용 부족 | 라벨링 데이터의 메타데이터가 위험도 판단에 충분히 활용되지 않는다. |

## 3. 사용자 정의

| 사용자 | 설명 | 주요 니즈 |
|---|---|---|
| 농장주 | 농작물 피해 예방이 필요한 사용자 | 야간 출현 위험 확인, 대응 방법 안내 |
| 산림관리원 | 산림·등산로 안전을 관리하는 사용자 | 야생동물 탐지 및 위험 상황 판단 |
| 지자체 담당자 | 주민 신고와 지역 안전을 관리하는 사용자 | 위험도 기록, 대응 안내, 관리 자료 확보 |
| 일반 사용자 | 야생동물을 발견한 사용자 | 안전한 행동 요령 확인 |

농장주, 산림관리원, 지자체 담당자를 포함하면 1차 ML, 2차 Vision, 3차 LLM, 4차 SLM이 모두 자연스럽게 연결된다.

## 4. 데이터 요구사항

### 4.1 1차 ML 데이터

| 항목 | 내용 |
|---|---|
| 원본 데이터 | AI Hub 야생동물 라벨링 JSON |
| 대상 폴더 | `data/aihub_json/TL-quadruped` |
| 변환 결과 | `data/aihub/ml_dataset/aihub_wildlife_metadata.csv` |
| 변환 행 수 | 30,700행 |
| 변환 컬럼 수 | 22개 |
| 주요 종 | 멧돼지 30,613건, 고라니 64건, 반달가슴곰 21건, 멧토끼 2건 |

주요 컬럼은 `file_name`, `date_created`, `hour`, `time_zone`, `season`, `day`, `camera_type`, `location`, `gps`, `weather`, `object_count`, `species`, `hazardous`, `nocturnality`, `max_bbox_area_ratio`, `avg_bbox_area_ratio`, `risk_level`이다.

### 4.2 위험도 라벨 생성 기준

원본 JSON에는 `risk_level`이 없으므로 객체 수, bbox 크기, 야간 여부를 기준으로 위험도 라벨을 생성한다.

| 조건 | 점수 |
|---|---:|
| 객체 수 3마리 이상 | +2 |
| 객체 수 2마리 | +1 |
| 최대 bbox 면적 비율 0.05 이상 | +2 |
| 최대 bbox 면적 비율 0.02 이상 | +1 |
| 시간대가 night 또는 dawn | +1 |
| day 값이 night | +1 |

| 점수 | 위험도 |
|---|---|
| 0~1 | low |
| 2~3 | medium |
| 4 이상 | high |

현재 위험도 분포는 medium 18,694건, high 10,196건, low 1,810건이다.

### 라벨 생성 이유
AI Hub 라벨링 JSON은 객체 위치, 동물 속성, 촬영 조건은 제공하지만 서비스용 위험도 정답 라벨은 제공하지 않는다. 따라서 객체 수, bbox 크기, 야간 여부를 활용하여 위험도 라벨을 생성한다.

## 5. 기능 요구사항

| ID | 요구사항 | 설명 |
|---|---|---|
| RQ-001 | JSON 메타데이터 변환 | AI Hub 라벨링 JSON을 1차 ML용 CSV로 변환한다. |
| RQ-002 | 위험도 라벨 생성 | 객체 수, bbox 면적, 시간대 등을 기반으로 risk_level을 생성한다. |
| RQ-003 | 위험도 예측 모델 학습 | 변환된 CSV를 사용하여 low / medium / high 위험도를 예측한다. |
| RQ-004 | 위험도 예측 화면 제공 | 사용자가 입력한 조건으로 위험도를 확인할 수 있다. |
| RQ-005 | YOLO 라벨 변환 | JSON bbox를 YOLO txt 형식으로 변환한다. |
| RQ-006 | 이미지 탐지 API | 이미지 업로드 시 사족보행 야생동물을 탐지한다. |
| RQ-007 | 상담 챗봇 | 위험도와 탐지 결과를 바탕으로 대응 방법을 안내한다. |
| RQ-008 | 결과 저장 | 예측 결과, 탐지 결과, 상담 로그를 Oracle DB에 저장한다. |
