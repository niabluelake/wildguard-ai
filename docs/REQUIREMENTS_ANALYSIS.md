# WildGuard AI 요구사항 분석서

## 1. 문서 목적

본 문서는 WildGuard AI 서비스의 요구사항을 정리하기 위한 문서이다. WildGuard AI는 야생동물 출몰 위험을 데이터 기반으로 예측하고, 이미지/영상 탐지 및 대응 상담을 제공하는 AI 기반 위험 대응 지원 서비스이다.

---

## 2. 서비스 개요

| 기능 | 설명 |
| --- | --- |
| 위험도 예측 | AI Hub 관측 지역, 날씨, 시간대, 계절 조건을 기반으로 야생동물 출몰 위험 점수 예측 |
| 이미지/영상 탐지 | YOLO 모델을 사용하여 이미지 또는 짧은 영상에서 야생동물 객체 탐지 |
| 대응 상담 챗봇 | 위험도 예측 결과를 바탕으로 사용자 질문에 대응 요령 안내 |
| 예측 기록 관리 | 로그인 사용자의 위험도 예측 결과를 DB에 저장하고 조회 |

---

## 3. 사용자 유형

| 사용자 | 목적 |
| --- | --- |
| 일반 사용자 | 특정 지역의 야생동물 출몰 위험도 확인 |
| 농장주 | 농장 주변 야생동물 출몰 위험과 대응 방법 확인 |
| 산림관리원 | 산림 및 국립공원 주변 위험 상황 확인 |
| 지자체 담당자 | 야생동물 출몰 위험 패턴과 예측 기록 확인 |
| 관리자 | 사용자, 예측 기록, 모델 결과 관리 |

---

## 4. 기능 요구사항

### 4.1 위험도 예측

- 사용자는 관측 지역, 날씨, 시간대, 계절을 입력할 수 있어야 한다.
- 시스템은 입력 조건을 기반으로 야생동물 출몰 위험 점수를 예측해야 한다.
- 시스템은 위험 점수를 `low`, `medium`, `high` 등급으로 변환해야 한다.
- 시스템은 주요 위험 동물, 과거 유사 관측 수, 권장 행동 요령을 제공해야 한다.
- 모델 파일이 없을 경우 명확한 오류 메시지를 반환해야 한다.

### 4.2 이미지/영상 탐지

- 사용자는 이미지 또는 짧은 영상을 업로드할 수 있어야 한다.
- 시스템은 YOLO 모델을 사용해 야생동물 객체를 탐지해야 한다.
- 시스템은 탐지된 객체 수, confidence, bbox 정보를 반환해야 한다.
- 시스템은 결과 이미지를 시각적으로 확인할 수 있어야 한다.

### 4.3 대응 상담 챗봇

- 사용자는 지역 조건과 질문을 입력할 수 있어야 한다.
- 시스템은 위험도 예측 결과를 바탕으로 상담 답변을 생성해야 한다.
- 상담 모드는 상세 상담 모드와 현장 대응 모드를 제공해야 한다.
- 상세 상담 모드는 위험도 해석, 위험 요인, 행동 요령을 충분히 설명해야 한다.
- 현장 대응 모드는 빠르게 읽을 수 있는 행동 체크리스트 중심으로 안내해야 한다.

### 4.4 회원 및 예측 기록

- 사용자는 회원가입 및 로그인을 할 수 있어야 한다.
- 로그인 사용자는 위험도 예측 결과를 DB에 저장할 수 있어야 한다.
- 로그인 사용자는 과거 예측 기록을 조회할 수 있어야 한다.
- 비회원도 위험도 예측 기능을 체험할 수 있어야 한다.

## 5. 비기능 요구사항

| 항목 | 요구사항 |
| --- | --- |
| 사용성 | 입력 폼과 결과 화면은 한글 기반으로 직관적으로 제공 |
| 확장성 | 향후 GPS, 기상 API, 신고 데이터 연동 가능 구조 |
| 유지보수성 | 라우트, 서비스, 템플릿, 스크립트를 기능별로 분리 |
| 보안 | 회원 비밀번호는 해시 처리 |
| 데이터 관리 | 원본 데이터와 모델 파일은 GitHub에 업로드하지 않음 |

## 7. 제약사항

- AI Hub 원본 데이터는 GitHub에 업로드하지 않는다.
- 학습된 모델 파일은 GitHub에 업로드하지 않는다.
- 현재 위험 점수는 실제 사고 피해 데이터가 아니라 관측 데이터 기반 설계 점수이다.
- 현재 위치 기반 실시간 예측이 아니라 AI Hub 관측 지역 기반 예측이다.
