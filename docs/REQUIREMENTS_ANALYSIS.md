# WildGuard AI 요구사항 분석서

## 1. 문서 목적

본 문서는 WildGuard AI 서비스의 요구사항을 정리하기 위한 문서이다.

WildGuard AI는 AI Hub 야생동물 관측 데이터와 한국도로공사 로드킬 다발 구간 데이터를 결합하여 야생동물 관측 상황의 현장 위험도를 평가하고, 이미지/영상 탐지 및 대응 상담을 제공하는 AI 기반 위험 대응 지원 서비스이다.

본 프로젝트의 1차 머신러닝 서비스는 단순 출몰 여부 예측이 아니라, 관측된 야생동물 상황을 기반으로 위험 점수, 위험 등급, 위험 요인, 현장 대응 가이드를 제공하는 현장 위험도 평가 서비스로 정의한다.

## 2. 서비스 개요

| 기능 | 설명 |
|---|---|
| 위험도 예측 | AI Hub 야생동물 관측 메타데이터와 로드킬 다발 구간 데이터를 기반으로 위험 점수 예측 |
| 위험 요인 설명 | 예측 결과에 영향을 준 주요 조건을 사용자에게 설명 |
| 현장 대응 가이드 | 위험 등급에 따른 행동 안내 메시지 제공 |
| 이미지/영상 탐지 | YOLO 모델을 사용하여 이미지 또는 짧은 영상에서 야생동물 객체 탐지 |
| 대응 상담 챗봇 | 위험도 예측 결과를 바탕으로 사용자 질문에 대응 요령 안내 |
| 예측 기록 관리 | 로그인 사용자의 위험도 예측 결과를 DB에 저장하고 조회 |

| 관리자 | 사용자, 예측 기록, 모델 결과 관리 |

## 4. 기능 요구사항

### 4.1 위험도 예측

- 사용자는 관측 지역, 날씨, 시간대, 계절을 입력할 수 있어야 한다.
- 시스템은 입력 조건과 기본 관측 시나리오를 기반으로 위험 점수를 예측해야 한다.
- 시스템은 위험 점수를 `low`, `medium`, `high` 등급으로 변환해야 한다.
- 시스템은 위험 요인 목록을 제공해야 한다.
- 시스템은 위험 등급에 따른 현장 대응 메시지를 제공해야 한다.
- 시스템은 로드킬 다발 구간 기반 외부 feature를 예측 결과와 함께 제공해야 한다.
- 모델 파일이 없을 경우 명확한 오류 메시지를 반환해야 한다.

### 4.2 로드킬 외부 feature 생성

- 시스템은 AI Hub 관측 데이터의 GPS 정보를 추출해야 한다.
- 시스템은 한국도로공사 로드킬 다발 구간 데이터의 위도·경도와 발생건수를 활용해야 한다.
- 시스템은 관측 지점과 로드킬 다발 구간 간 거리를 계산해야 한다.
- 시스템은 반경 5km, 10km, 20km 내 로드킬 발생건수를 집계해야 한다.
- 시스템은 거리와 발생건수를 반영한 `roadkill_weighted_score`를 생성해야 한다.
- 시스템은 생성된 로드킬 feature를 최종 학습 데이터셋에 결합해야 한다.

### 4.3 이미지/영상 탐지

- 사용자는 이미지 또는 짧은 영상을 업로드할 수 있어야 한다.
- 시스템은 YOLO 모델을 사용해 야생동물 객체를 탐지해야 한다.
- 시스템은 탐지된 객체 수, confidence, bbox 정보를 반환해야 한다.
- 시스템은 결과 이미지를 시각적으로 확인할 수 있어야 한다.

### 4.4 대응 상담 챗봇

- 사용자는 지역 조건과 질문을 입력할 수 있어야 한다.
- 시스템은 위험도 예측 결과를 바탕으로 상담 답변을 생성해야 한다.
- 상담 모드는 상세 상담 모드와 현장 대응 모드를 제공해야 한다.
- 상세 상담 모드는 위험도 해석, 위험 요인, 행동 요령을 충분히 설명해야 한다.
- 현장 대응 모드는 빠르게 읽을 수 있는 행동 체크리스트 중심으로 안내해야 한다.

### 4.5 회원 및 예측 기록

- 사용자는 회원가입 및 로그인을 할 수 있어야 한다.
- 로그인 사용자는 위험도 예측 결과를 DB에 저장할 수 있어야 한다.
- 로그인 사용자는 과거 예측 기록을 조회할 수 있어야 한다.
- 비회원도 위험도 예측 기능을 체험할 수 있어야 한다.

| 항목 | 요구사항 |

## 7. 위험도 예측 데이터 컬럼

### 7.1 기본 관측 feature

| 컬럼 | 설명 |
|---|---|
| `day` | 주간/야간 |
| `camera_type` | IR/RGB 카메라 유형 |
| `weather` | 날씨 |
| `location` | 관측 위치 |
| `time_zone` | 시간대 |
| `season` | 계절 |
| `species` | 관측 종 |
| `object_count` | 객체 수 |
| `max_bbox_area_ratio` | 가장 큰 bbox 면적 비율 |
| `avg_bbox_area_ratio` | 평균 bbox 면적 비율 |

### 7.2 GPS 및 로드킬 feature

| 컬럼 | 설명 |
|---|---|
| `latitude` | AI Hub 관측 지점 위도 |
| `longitude` | AI Hub 관측 지점 경도 |
| `nearest_roadkill_distance_km` | 가장 가까운 로드킬 다발 구간까지 거리 |
| `roadkill_count_within_5km` | 반경 5km 내 로드킬 발생건수 |
| `roadkill_count_within_10km` | 반경 10km 내 로드킬 발생건수 |
| `roadkill_count_within_20km` | 반경 20km 내 로드킬 발생건수 |
| `roadkill_max_cases_nearby` | 주변 로드킬 다발 구간 중 최대 발생건수 |
| `roadkill_weighted_score` | 거리와 발생건수를 반영한 로드킬 가중 점수 |
| `near_roadkill_hotspot` | 10km 내 로드킬 hotspot 존재 여부 |

## 8. 제약사항

- AI Hub 원본 데이터는 GitHub에 업로드하지 않는다.
- 한국도로공사 로드킬 원본 및 통합 CSV는 GitHub에 업로드하지 않는다.
- 학습된 모델 파일은 GitHub에 업로드하지 않는다.
- 현재 위험 점수는 실제 사고 피해 데이터가 아니라 관측 데이터와 외부 공공데이터 기반 설계 점수이다.
- 현재 웹 화면은 사용자가 직접 GPS를 입력하는 방식이 아니라, AI Hub 관측 지역과 기본 관측 시나리오를 기준으로 위험도를 확인한다.
- 실제 위치 기반 실시간 로드킬 feature 계산은 향후 개선 항목으로 둔다.

---

## 9. 향후 개선 요구사항

- 사용자가 입력한 실제 GPS 좌표 기반 실시간 로드킬 feature 계산
- 기상청 API 기반 실시간 날씨 feature 추가
- 토지피복도 및 생태자연도 기반 공간환경 feature 추가
- 위험도 예측 결과와 YOLO 탐지 결과 자동 연동
- 예측 기록 기반 지역별 위험 통계 대시보드 구현
- 지도 기반 위험 hotspot 시각화
