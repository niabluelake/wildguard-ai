# WildGuard AI PRD

## 1. 제품 개요

WildGuard AI는 야생동물 관측 상황의 위험도를 평가하고, 이미지/영상 탐지와 대응 상담을 통해 사용자에게 실질적인 행동 요령을 제공하는 AI 기반 위험 대응 지원 서비스이다.

본 서비스는 AI Hub 야생동물 관측 데이터와 한국도로공사 로드킬 다발 구간 데이터를 결합하여, 단순 출몰 예측이 아니라 현장 상황 기반 위험도 평가를 제공한다.

---

## 2. 문제 정의

농장, 산림, 국립공원, 마을 주변에서는 야생동물 출몰로 인한 안전 문제와 시설물 피해 가능성이 존재한다.

그러나 일반 사용자는 현재 조건에서 위험도가 높은지, 어떤 요소가 위험도를 높였는지, 실제로 어떤 행동을 해야 하는지 판단하기 어렵다.

또한 기존의 단순 위험 등급만 제공하는 방식은 사용자가 결과를 신뢰하거나 행동으로 연결하기 어렵다.

WildGuard AI는 관측 데이터 기반 위험도 예측, 로드킬 다발 구간 외부 feature, 이미지 탐지, 대응 상담 기능을 결합하여 사용자의 의사결정을 지원한다.

---

## 3. 제품 목표

- AI Hub 야생동물 관측 데이터를 활용한 위험도 예측
- 한국도로공사 로드킬 다발 구간 데이터를 외부 feature로 결합
- 위험 점수, 위험 등급, 위험 요인, 대응 메시지를 함께 제공
- YOLO 기반 이미지/영상 야생동물 탐지
- 위험도 예측 결과 기반 대응 상담 제공
- 로그인 사용자의 예측 기록 저장 및 조회
- 향후 GPS, 신고 데이터, 실시간 기상 데이터와 연동 가능한 구조 설계

## 4. 핵심 사용자

| 사용자 | 니즈 |
|---|---|
| 농장주 | 농장 주변 야생동물 출현 위험과 대응 방법 확인 |
| 산림관리원 | 산림 및 국립공원 주변 위험 상황 파악 |
| 일반 사용자 | 주변 지역의 야생동물 위험도 확인 |

| 관리자 | 서비스 사용자와 예측 기록 관리 |

---

## 5. 핵심 사용자 시나리오

### 시나리오 1. 지역 위험도 확인

1. 사용자가 `/risk` 화면에 접속한다.
2. 관측 지역, 날씨, 시간대, 계절을 선택한다.
3. 시스템이 기본 관측 시나리오와 외부 로드킬 feature를 기반으로 위험 점수를 예측한다.
4. 사용자는 위험 점수와 위험 등급을 확인한다.
5. 사용자는 주요 위험 요인과 현장 대응 가이드를 확인한다.
6. 사용자는 로드킬 다발 구간 기반 외부 feature를 통해 위험 판단 근거를 확인한다.

### 시나리오 2. 이미지/영상 탐지

1. 사용자가 `/vision` 화면에 접속한다.
2. 이미지 또는 짧은 영상을 업로드한다.
3. 시스템이 YOLO 모델로 야생동물을 탐지한다.
4. 사용자는 bbox, confidence, 결과 이미지를 확인한다.
5. 탐지 결과는 향후 위험도 예측 결과와 통합할 수 있다.

### 시나리오 3. 대응 상담

1. 사용자가 `/chat` 화면에 접속한다.
2. 지역 조건과 질문을 입력한다.
3. 시스템이 위험도 예측 결과를 바탕으로 상담 답변을 생성한다.
4. 사용자는 상세 상담 또는 현장 대응 안내를 확인한다.

## 6. 주요 기능

| 기능 | 사용자 가치 |
|---|---|
| 위험도 예측 | 특정 조건에서 야생동물 관측 상황의 위험도를 정량적으로 확인 |
| 위험 요인 설명 | 왜 위험도가 높거나 낮은지 이해 |
| 현장 대응 가이드 | 위험 상황에서 무엇을 해야 하는지 확인 |
| 로드킬 외부 feature | 주변 과거 야생동물 사고 다발 구간 정보를 위험도에 반영 |
| 이미지/영상 탐지 | 실제 촬영 자료에서 야생동물 존재 여부 확인 |
| 대응 상담 | 위험 상황에서 구체적인 행동 요령 안내 |
| 예측 기록 | 과거 예측 결과를 저장하고 다시 확인 |

## 7. 성공 기준

| 항목 | 기준 |
|---|---|
| 위험도 예측 API | 정상 입력 시 위험 점수, 등급, 위험 요인, 대응 메시지 반환 |
| 로드킬 feature 결합 | GPS 기반 로드킬 거리 feature가 최종 학습 데이터셋에 포함 |
| 모델 학습 | `risk_regression_model.pkl` 생성 가능 |
| 웹 화면 | `/risk`에서 위험 점수, 위험 요인, 대응 메시지, 로드킬 feature 표시 |
| 이미지 탐지 | `/vision`에서 이미지 업로드 및 탐지 결과 확인 가능 |
| 상담 API | 질문 입력 시 위험도 기반 답변 반환 |
| Git 관리 | 데이터와 모델 파일 제외, 코드와 문서만 관리 |

## 8. 핵심 지표

| 지표 | 설명 |
|---|---|
| API 성공 여부 | `/api/risk/predict` 호출 시 `success: true` 반환 |

---

## 9. 현재 모델 성능

로드킬 feature를 결합한 최종 회귀 모델 기준 성능은 다음과 같다.

| 지표 | 값 |
|---|---:|
| MAE | 0.0276 |
| RMSE | 0.2472 |
| R2 Score | 0.9991 |

해당 성능은 실제 사고 발생 여부의 정확도가 아니라, 설계한 위험 점수 체계를 회귀 모델이 학습한 정도를 의미한다.

---

## 10. 현재 제약사항

- 현재 위험 점수는 실제 사고 데이터가 아니라 관측 데이터와 외부 공공데이터 기반 설계 점수이다.
- 현재 웹 화면은 사용자가 직접 GPS를 입력하는 방식이 아니라 AI Hub 관측 지역 기반으로 동작한다.
- 로드킬 데이터는 개별 사고 로그가 아니라 고속도로 구간별 다발 지점 데이터이다.
- 데이터 분포상 멧돼지 비중이 매우 높아 주요 위험 종이 멧돼지로 편향될 수 있다.
- 모델과 원본 데이터는 GitHub에 업로드하지 않는다.

---

## 11. 향후 개선

- GPS 기반 현재 위치 위험도 예측
- 기상청 API 기반 실시간 날씨 feature 추가
- 토지피복도 및 생태자연도 기반 공간환경 feature 추가
- 지역별 신고 데이터 및 피해 기록 연동
- 위험 지도 시각화
- 이미지 탐지 결과와 위험도 예측 결과 통합
- 예측 기록 기반 지역별 위험 통계 대시보드 구현
- 상담 답변 고도화
- 실제 피해 데이터 확보 시 위험 점수 체계 재설계
