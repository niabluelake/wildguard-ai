# WildGuard AI PRD

## 1. 문서 개요

### 문서명

WildGuard AI 제품 요구사항 문서

### 프로젝트명

WildGuard AI

### 제품 목적

WildGuard AI는 AI Hub 야생동물 라벨링 데이터를 기반으로 야생동물 관측 상황의 위험 점수를 예측하고, 이미지 탐지와 대응 상담 기능으로 확장하는 AI 기반 야생동물 대응 지원 서비스이다.

본 PRD는 1차 머신러닝 프로젝트를 중심으로, 향후 2차 Vision, 3차 LLM, 4차 SLM으로 확장 가능한 제품 요구사항을 정의한다.

---

## 2. 제품 개요

### 제품 한 줄 설명

야생동물 관측 조건을 입력하면 위험 점수를 예측하고, 위험 등급과 대응 메시지를 제공하는 AI 서비스

### 핵심 기능

| 구분        | 기능                            |
| --------- | ----------------------------- |
| 1차 ML     | AI Hub JSON 메타데이터 기반 위험 점수 예측 |
| 2차 Vision | 이미지 기반 야생동물 탐지                |
| 3차 LLM    | 위험 상황 대응 상담                   |
| 4차 SLM    | 경량 현장 대응 상담                   |

### 현재 구현 범위

현재 구현된 1차 ML 기능은 AI Hub 라벨링 JSON을 CSV로 변환하고, `risk_score`를 생성한 뒤, `RandomForestRegressor` 회귀 모델로 위험 점수를 예측하는 구조이다.

---

## 3. 문제 정의

야생동물 출현은 농작물 피해, 시설물 파손, 인명 안전 문제를 유발할 수 있다. 그러나 일반 사용자는 동물 종류, 촬영 시간대, 날씨, 객체 수, 동물과의 거리감 등을 종합하여 위험도를 판단하기 어렵다.

특히 단순히 `low`, `medium`, `high` 등급만 제공하면 위험 수준의 세부 차이를 표현하기 어렵다.

따라서 WildGuard AI는 야생동물 관측 상황을 0~100 범위의 위험 점수로 수치화하고, 해당 점수에 따라 위험 등급과 대응 메시지를 제공한다.

## 6. 사용자 시나리오

### 시나리오 1. 농장주 야간 멧돼지 관측

1. 농장주가 야간에 IR 카메라로 촬영된 멧돼지 관측 정보를 입력한다.
2. 시스템은 동물 종, 시간대, 날씨, 객체 수, bbox 면적 비율을 기반으로 위험 점수를 예측한다.
3. 위험 점수가 높게 나오면 `high` 등급과 함께 시설물 점검 및 현장 접근 주의 메시지를 제공한다.

### 시나리오 2. 산림관리원 반복 출현 모니터링

1. 산림관리원이 산림 지역에서 반복 관측된 야생동물 정보를 입력한다.
2. 시스템은 위험 점수와 등급을 반환한다.
3. 산림관리원은 결과를 바탕으로 현장 점검 필요 여부를 판단한다.

### 시나리오 3. 향후 이미지 탐지 연동

1. 사용자가 야생동물 사진을 업로드한다.
2. YOLO 모델이 이미지에서 야생동물을 탐지한다.
3. 탐지 결과와 1차 위험 점수 예측 결과를 결합하여 대응 상담으로 확장한다.

## 9. 위험 점수 설계

### 설계 배경

AI Hub 원본 JSON에는 실제 사고 피해 점수나 서비스용 위험도 점수가 포함되어 있지 않다.

따라서 본 프로젝트에서는 야생동물 관측 상황을 정량화하기 위해 도메인 가중치 기반의 `risk_score`를 설계한다.

### 반영 요소

| 요소          | 반영 이유                             |
| ----------- | --------------------------------- |
| 동물 종        | 종에 따라 사람, 농작물, 시설물에 미치는 위험도가 다르다. |
| 시간대         | 야간과 새벽은 시야 확보와 대응이 어렵다.           |
| 날씨          | 비, 눈 등은 현장 대응 난도를 높일 수 있다.        |
| 객체 수        | 여러 개체가 동시에 관측되면 위험도가 높아질 수 있다.    |
| bbox 면적 비율  | 화면에서 크게 잡힌 객체는 카메라와 가까울 가능성이 있다.  |
| IR 야간 촬영 여부 | 야간 출현 상황일 가능성이 높다.                |

### 점수 범위

```text
risk_score = 0 ~ 100
```

### 등급 기준

| risk_score   | risk_grade |
| ------------ | ---------- |
| 0 이상 45 미만   | low        |
| 45 이상 70 미만  | medium     |
| 70 이상 100 이하 | high       |

### 현재 위험 등급 분포

| risk_grade |  count |
| ---------- | -----: |
| medium     | 19,685 |
| high       | 10,343 |
| low        |    672 |

## 10. 모델 요구사항

### 모델 유형

지도학습 기반 회귀 모델

### 사용 모델

RandomForestRegressor

### 입력 Feature

| 구분  | 컬럼                                                                            |
| --- | ----------------------------------------------------------------------------- |
| 범주형 | `day`, `camera_type`, `weather`, `location`, `time_zone`, `season`, `species` |
| 수치형 | `object_count`, `max_bbox_area_ratio`, `avg_bbox_area_ratio`                  |

### Target

```text
risk_score
```

### 저장 파일

```text
models/ml/risk_regression_model.pkl
```

### 모델 성능

| 지표       |      값 |
| -------- | -----: |
| MAE      | 0.0183 |
| RMSE     | 0.2324 |
| R2 Score | 0.9992 |

### 성능 해석

현재 `risk_score`는 실제 사고 피해 데이터가 아니라 도메인 가중치 기반으로 설계한 위험 점수이다.

따라서 위 성능은 실제 사고 위험을 거의 완벽하게 예측했다는 의미가 아니라, 설계된 위험 점수 체계를 회귀 모델이 얼마나 잘 학습했는지를 의미한다.

향후 실제 신고 데이터, 피해 기록, 기상 데이터, 지역 환경 데이터가 확보되면 현재의 `risk_score`를 실제 데이터 기반 라벨로 교체할 수 있다.

## 17. 주요 리스크

| 리스크         | 설명                           | 대응 방안                 |
| ----------- | ---------------------------- | --------------------- |
| 실제 사고 라벨 부재 | AI Hub JSON에는 실제 피해 점수가 없음   | 도메인 가중치 기반 초기 점수임을 명시 |
| 종 분포 불균형    | 멧돼지 데이터가 대부분을 차지함            | 향후 데이터 보강 또는 샘플링 고려   |
| 모델 성능 오해    | R2가 높아도 실제 사고 예측 성능은 아님      | 성능 해석 문서화             |
| 모델 파일 용량    | pkl 파일이 약 98MB로 큼            | GitHub 제외, 로컬 재생성     |
| 데이터 라이선스    | AI Hub 원본 데이터 업로드 불가         | 원본 데이터는 로컬 보관         |
| 웹 화면 불일치    | API는 점수 기반이나 UI가 등급 중심일 수 있음 | `risk.html` 수정 필요     |
